What is the Document Object Mode (DOM)?

The Document Object Model (DOM) is a programming interface for representing HTML, XML, and other structured documents as a tree of nodes. Each node represents a part of the document, such as an element, attribute, text node, or comment. The DOM provides a way for developers to access and manipulate the structure, content, and style of a document.

What is event delegation in the context of the DOM, and why is it useful?

Event Delegation in the DOM
Event delegation is a technique in JavaScript where you attach an event listener to a parent element instead of directly to each child element. This allows you to handle events for multiple child elements efficiently, without having to add individual event listeners to each one.

Why is it useful?
Improved performance:
When you attach event listeners to individual elements, the browser has to create and maintain a separate event listener object for each element. This can lead to performance issues, especially when dealing with a large number of elements. With event delegation, you only need to create one event listener for the parent element.

Dynamic content:
Event delegation is particularly useful when dealing with dynamically generated content. You can attach an event listener to a parent element once, and it will automatically handle events for any child elements that are added later.

Simplified code:
Event delegation can simplify your code by reducing the amount of repetitive code needed to handle events for multiple elements.

How do you manipulate an element's attributes and styles using the DOM?

The DOM provides methods to access and modify an element's attributes and styles.
Accessing and Modifying Attributes
getAttribute(attributeName): Retrieves the value of the specified attribute.
setAttribute(attributeName, value): Sets the value of the specified attribute.
removeAttribute(attributeName): Removes the specified attribute.
